# /code-review

コードレビューを実行するコマンド。

## 実行手順

1. **変更ファイル取得**
```bash
git diff --name-only HEAD
```

2. **レビュー実施**

### セキュリティ（ブロッキング）
- [ ] ハードコードされた認証情報
- [ ] インジェクション脆弱性
- [ ] 安全でない依存関係

### 品質（高優先度）
- [ ] 関数サイズ（50行以下）
- [ ] ファイルサイズ（800行以下）
- [ ] ネスト深度（4段階以下）
- [ ] エラーハンドリング
- [ ] console.log残存

### ベストプラクティス（中優先度）
- [ ] コードパターン
- [ ] テストカバレッジ
- [ ] アクセシビリティ

3. **結果出力**

```markdown
## コードレビュー結果

### CRITICAL
- なし

### HIGH
- [ファイル:行] - 問題と修正提案

### MEDIUM
- [ファイル:行] - 問題と修正提案

### LOW
- [ファイル:行] - 問題と修正提案

## 判定: APPROVE / WARNING / BLOCK
```

## 判定基準

- **APPROVE**: Critical/High問題なし
- **WARNING**: Medium問題のみ
- **BLOCK**: Critical/High問題あり

## 注意

Critical/High問題がある場合は絶対に承認しない。
