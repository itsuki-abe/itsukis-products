# /plan

実装計画を作成するコマンド。

## 実行手順

1. **要件を明確化**
   - 何を作るのか
   - なぜ必要なのか
   - 制約事項は何か

2. **既存コード調査**
   - 関連ファイルを特定
   - 既存パターンを理解
   - 影響範囲を把握

3. **計画作成**

```markdown
# [機能名] 実装計画

## 概要
[1-2文で説明]

## 要件
### 機能要件
-

### 非機能要件
-

## 影響範囲
### 新規ファイル
-

### 変更ファイル
-

## フェーズ

### Phase 1: [名前]
**目標**: ...
**ファイル**:
- [ ] `path/to/file.ts` - [変更内容]

### Phase 2: [名前]
...

## リスク
| リスク | 重要度 | 対策 |
|--------|--------|------|
|        |        |      |

## テスト戦略
- ユニットテスト:
- 統合テスト:
- E2Eテスト:
```

4. **ユーザー確認**
   - 計画を提示
   - 承認を得てから実装開始

## 重要

**コードは書かない**: 計画が承認されるまで実装しない

## 次のステップ

承認後:
- `/tdd` - テスト駆動で実装
- `/build-fix` - ビルドエラー修正
- `/code-review` - コードレビュー
